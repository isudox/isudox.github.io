M 记对代码质量要求很高，有两个指标是卡的很严的，一个是单元测试的覆盖率，另一个是干行代码 bug 率。组里对单测有过专门的讨论，从经验上看，如果要达到较好的覆盖率，业务代码和测试代码的耗时接近 1:1 的关系，同事建议考虑采用 Spock 代替 JUnit 作为单测框架，能提高不少效率。之前没有接触过 Spock，但初上手，确实能感觉到便利。
